About & GitHub

Quick answer: This is an unofficial mirror and installer repository for LibreWolf Windows builds. Source code and packaging scripts are available on GitHub — contributions, issue reports and packaging improvements are welcome. Always verify checksums before installing.

Repository & sources

Primary code and packaging scripts used to build the Windows binaries are hosted on GitHub. For transparency, we link upstream releases and include manifests where possible. If you maintain a fork or packaging script, feel free to open a PR or contact the repo owner.

How releases are built (high level)

  1. Fetch upstream Firefox source and LibreWolf patches.
  2. Apply packaging patches and build scripts for Windows targets.
  3. Run CI to produce artifacts, sign or checksum binaries, and publish to mirrors.
  4. Update site content (download links, checksums, release notes) and update sitemap.

Contributing

We welcome contributions to packaging scripts, mirrors, translations and documentation. Typical contribution flow:

  • Fork the repository on GitHub.
  • Create a feature branch and submit a pull request with clear descriptions and tests where applicable.
  • Maintain semantic commit messages and reference issue numbers for easier review.

Security & provenance

Always verify checksums and signatures. We publish SHA256 checksums on the Download page but do not host PGP keys centrally — check the upstream project or official mirrors for signing keys. If you discover a security issue, open a private issue in the GitHub repo or contact the maintainer according to the repository's security policy.

Contact & reporting issues

For packaging issues, opensource contributions, or mirror problems, open an issue on the GitHub repository. For security-sensitive reports, follow the repository's security disclosure instructions.
